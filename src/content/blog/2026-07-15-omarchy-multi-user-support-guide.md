モダンなLinuxデスクトップ環境、特にWaylandコンポジタの「Hyprland」や直感的な「QuickShell」などを組み合わせた環境は、近年多くの開発者から注目を集めています。その中でも、あらかじめ最適化された設定を提供する「おまかせ（Omakase）」思想を掲げるシステム「Omarchy」は、洗練されたユーザー体験を即座に得られることで人気を博しています。

しかし、Omarchyはその設計思想上、**「1台のPCを1人のユーザーが専有して使用する（シングルユーザー）」**ことを前提に最適化されています。そのため、家族や同僚とPCを共有するために「複数ユーザー（マルチユーザー）アカウント」をセットアップしようとすると、いくつかの技術的なハードルに直面します。

今回は、Redditに寄せられた質問「Omarchyでマルチユーザーサポートを有効にするには？」をもとに、Omarchyの設計背景を紐解きながら、マルチユーザー環境を構築するための具体的なアプローチと注意点を解説します。

---

## なぜOmarchyは「シングルユーザー」前提なのか？

Omarchyがシングルユーザー向けに設計されている最大の理由は、**「摩擦のない極上のデスクトップ体験」**を追求している点にあります。

具体的には、以下のような挙動がデフォルトで組み込まれています。

1. **シームレスな自動ログイン（Auto-login）：**
   PCの電源を入れてからデスクトップ（Hyprlandなど）が立ち上がるまでに、ディスプレイマネージャー（ログイン画面）を挟まず、特定のユーザーで自動ログインするサービスが有効になっています。
2. **設定ファイル（Dotfiles）の強固な紐付け：**
   システム全体の設定と、ユーザー個人のホームディレクトリ配下にある設定（`~/.config/hypr` や `~/.config/quickshell` など）が密接に連携しており、単一ユーザーが使用したときに最もパフォーマンスを発揮するようにチューニングされています。

この「おまかせ」の快適さを維持したまま、別のアカウントを追加して環境を切り替えるには、いくつかのシステム設定を手動で調整する必要があります。

## Omarchyでマルチユーザー環境を実現するための4つのステップ

Omarchyで複数ユーザーをサポートするためには、自動ログインを無効化し、ユーザーごとにセッションを管理できる「ディスプレイマネージャー（ログインランチャー）」を適切に導入・設定する必要があります。

### ステップ1：自動ログイン（Auto-login）の無効化

Omarchyは、SystemdサービスやGetty、あるいは`greetd`などの軽量なディスプレイマネージャーを利用して自動ログインを実現しています。まずはこの自動ログイン設定を解除します。

例えば、`greetd`（多くのモダンなWayland環境で採用されているログインマネージャー）が使用されている場合は、`/etc/greetd/config.toml` を確認します。

```toml
# /etc/greetd/config.toml の例
[initial_session]
command = "Hyprland"
user = "your_default_user"  # ← この自動ログイン設定をコメントアウトまたは削除します
```

自動ログインを無効化することで、起動時にユーザー選択とパスワード入力を求めるプロンプト（またはログイン画面）が表示されるようになります。

### ステップ2：新規ユーザーの追加と権限付与

Linuxの標準的なコマンドを使用して、新しいユーザーを追加します。

```bash
sudo useradd -m -G wheel,video,audio,input -s /bin/bash newuser
sudo passwd newuser
```

*※ `wheel` グループに追加することで、新ユーザーにも `sudo` 権限を付与できます（必要に応じて制限してください）。また、Wayland（Hyprland）を適切に動作させるために、`video` や `input` グループへの所属が必要になる場合があります。*

### ステップ3：ディスプレイマネージャーの調整（TUI/GUIログインの導入）

マルチユーザー環境では、起動時にユーザーを選択できるインターフェースが必要です。Hyprland環境と相性が良く、軽量な **`tuigreet`**（greetdのコンソール向けフロントエンド）や、グラフィカルな **`SDDM`**（Waylandサポート版）の導入を推奨します。

`greetd` と `tuigreet` を使用する場合、`/etc/greetd/config.toml` を以下のように設定します。

```toml
[default_session]
command = "tuigreet --cmd Hyprland --time --remember --remember-session"
user = "greeter"
```

これにより、起動時に美しいTUI（テキストユーザインターフェース）のログイン画面が表示され、ユーザー名とパスワードを入力してそれぞれの環境にログインできるようになります。

### ステップ4：新ユーザーへのOmarchy設定（Dotfiles）の適用

新規作成したユーザーのホームディレクトリ（`/home/newuser/`）は、空っぽか、あるいはLinuxのデフォルトのスケルトンファイル（`/etc/skel`）のみが存在する状態です。このままでは、Omarchyの美しいデスクトップ環境（HyprlandやQuickShellの設定）が適用されません。

メインユーザーの設定ファイル群（`~/.config` 配下など）を新ユーザーのディレクトリにコピーし、所有権を適切に変更する必要があります。

```bash
# 例：メインユーザーの設定を新ユーザーにコピー
sudo cp -r /home/mainuser/.config /home/newuser/
sudo chown -R newuser:newuser /home/newuser/.config
```

*注意：設定ファイル内にメインユーザーのユーザー名（パスなど）がハードコードされている場合、新ユーザーの環境でエラーが発生することがあります。コピー後は、設定ファイル内のパスを適宜修正してください。*

## マルチユーザー化における注意点とデメリット

Omarchyをマルチユーザー化する際には、以下のデメリットやリスクを考慮する必要があります。

* **アップデート時の競合：**
  Omarchyはシステムアップデート時に、デフォルトユーザーの設定ファイルを自動で更新・同期するスクリプトを持っている場合があります。手動で追加した複数ユーザーのDotfilesは、これらの自動更新の対象外になるか、あるいはアップデートによってカスタマイズが上書きされてしまう可能性があります。
* **ディスク容量の消費：**
  ユーザーごとにキャッシュや設定ファイル、ブラウザのプロファイルが作成されるため、ディスク容量を圧迫します。
* **「おまかせ」体験の崩壊：**
  Omarchyの魅力は「開発者が用意した最高の環境に、電源を入れて数秒でアクセスできること」です。ログイン画面を挟むことで、そのシームレスな体験は一歩後退することになります。

## まとめ：自分だけの環境か、共有の利便性か

Omarchyでマルチユーザー環境を構築することは、Linuxの標準的な知識（Systemd、ディスプレイマネージャー、ユーザー権限の管理）があれば十分に可能です。

しかし、Omarchy本来の「極限まで無駄を削ぎ落としたシングルユーザー向けおまかせ環境」というコンセプトを考慮すると、もし頻繁に複数人でPCを共有するのであれば、一般的なArch Linuxに手動でHyprlandを導入するか、あるいはマルチユーザーを最初から想定している他のディストリビューションを検討するのも一つの手です。

「どうしてもOmarchyの洗練されたプリセットを複数人で使いたい」という方は、ぜひ本記事のアプローチを参考に、バックアップを入念に取った上でカスタマイズに挑戦してみてください。
