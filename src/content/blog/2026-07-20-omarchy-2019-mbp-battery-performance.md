近年、Apple Silicon（M1/M2/M3/M4）への移行に伴い、一世代前となったIntelプロセッサ搭載のMacBook ProにLinuxをインストールし、開発環境として再利用するアプローチが注目を集めています。

特に、Arch LinuxをベースにHyprlandなどのモダンなWayland環境を「おまかせ（Omakase）」で構築できるメタディストリビューション**「Omarchy」**は、軽量かつスタイリッシュなデスクトップ環境を求めるパワーユーザーに人気です。

しかし、Intel Mac上でLinuxを運用する際、避けて通れないのが**「バッテリー持続時間（バッテリーライフ）」の課題**です。今回は、Redditの「r/omarchy」コミュニティに寄せられた2019年製MacBook Pro（i7/i9モデル）のバッテリー性能に関する疑問をフックに、Intel MacにおけるLinuxの電力管理の現状と、実用的な最適化手法について専門的な視点から解説します。

---

## 2019年製MacBook Pro（Intel Mac）とLinuxの相性

Redditの投稿では、2019年製MacBook Pro（14インチまたは16インチ、i7またはi9構成）におけるOmarchyのバッテリーライフについて質問されています。

まず技術的な補足として、**2019年モデルのMacBook Proのラインナップは「13インチ」「15インチ」「16インチ」**であり、14インチモデルが登場したのはApple Silicon（M1 Pro/Max）以降の2021年です。したがって、投稿者が言及しているのはおそらく15インチまたは16インチの、独立GPU（dGPU）を搭載した上位モデル（i7/i9）を指していると考えられます。

これらのモデルでLinuxを動作させる場合、以下の2つの大きな技術的ハードルが存在します。

1. **T2セキュリティチップの壁**
   2018年以降のIntel Macには「Apple T2 セキュリティチップ」が搭載されており、キーボード、トラックパッド、オーディオ、SSDコントローラーなどを制御しています。これらをLinuxで正常に動作させるには、有志による[Linux on T2 Macs](https://t2linux.org/)プロジェクトが提供するカスタムカーネル（`linux-t2`）の導入が必須となります。
2. **ハイブリッドグラフィックス（iGPU / dGPU）の制御**
   15/16インチモデルには、CPU内蔵のIntelグラフィックス（iGPU）と、強力なAMD Radeonグラフィックス（dGPU）の両方が搭載されています。macOSでは負荷に応じてこれらがシームレスに切り替わりますが、Linuxのデフォルト状態ではdGPUが常時稼働してしまい、バッテリーを激しく消費する原因になります。

## なぜLinux環境ではバッテリー消費が激しいのか？

macOSは、AppleがハードウェアとOSを垂直統合して開発しているため、極めて高度な電力最適化が施されています。一方、汎用OSであるLinuxをMacBook Proに導入した場合、以下のような理由からバッテリー消費が大幅に増加する傾向にあります。

- **dGPUの常時通電**: 画面出力がdGPUに固定されてしまうと、アイドル時でも10W〜20W以上の電力を消費し続けます。
- **CPUのクロック制御**: Intel Core i7/i9（特に第9世代）は非常に発熱しやすく、消費電力も大きいです。LinuxのデフォルトのCPUガバナー（省電力制御）がMacBookの排熱機構と噛み合わない場合、不必要にファンが回り、クロックが高止まりします。
- **Wayland/Hyprlandの描画負荷**: Omarchyが採用しているHyprlandは非常に軽量なタイル型コンポジタですが、ハードウェアアクセラレーション（GPU描画）やリッチなアニメーション効果を使用するため、設定次第ではGPUの消費電力を引き上げる要因になります。

## Omarchy環境でバッテリーライフを最大化する最適化手法

もし2019年製MacBook ProにOmarchyをインストールしてモバイル環境で活用する場合、以下の最適化を行うことで、バッテリー持続時間を大幅に改善（実用レベルの3〜5時間程度へ向上）させることが可能です。

### 1. グラフィックスの切り替え（dGPUの無効化）
モバイル用途（ブラウジング、テキストエディタ、ターミナル作業など）がメインであれば、AMD Radeon（dGPU）を完全に無効化し、Intel iGPUのみで動作させるのが最も効果的です。

Arch Linux環境では、`supergfxctl` や `envycontrol` といったツールを利用して、グラフィックスモードを「Integrated（内蔵グラフィックスのみ）」に切り替えることができます。これにより、dGPUへの給電がカットされ、消費電力を一気に削減できます。

### 2. `auto-cpufreq` または `TLP` の導入
Linuxの省電力ツールを導入し、バッテリー駆動時のCPU動作クロックやシステムの給電状態を自動で最適化します。

- **`auto-cpufreq`**: CPUの負荷や電源接続状態を監視し、CPUのガバナー（performance / powersave）やターボブーストのオン/オフを動的に調整してくれるため、Intel i7/i9の爆熱と電力消費を抑えるのに非常に有効です。
- **`TLP`**: 伝統的なLinux用省電力ツールで、PCI Expressデバイス、USBポート、SATAリンクなどの細かな省電力設定を一括で行ってくれます。

*注意: `auto-cpufreq` と `TLP` を同時に動かすと競合が発生する場合があるため、片方をメインに据えて調整することをおすすめします。*

### 3. HyprlandのVFR（可変フレームレート）の有効化
OmarchyのコアであるHyprlandの設定ファイル（`hyprland.conf`）において、画面に変化がないときの描画回数を減らす**VFR（Variable Frame Rate）**を有効にします。

```ini
misc {
    vfr = true
}
```

これに加え、バッテリー駆動時にはアニメーション（`animations`）を一時的にオフにするスクリプトを導入するのも効果的です。

## まとめ：実用的なモバイル環境としての評価

2019年製MacBook Pro（i7/i9）上でOmarchyを動かす場合、未対策のデフォルト状態では**「バッテリーが1時間半〜2時間しか持たない」**という状況に陥りがちです。

しかし、上記で紹介した**「dGPUの無効化（iGPU固定）」「auto-cpufreqによるCPU制御」「Hyprlandの描画最適化」**を施すことで、軽作業であれば**3時間半〜5時間程度**まで引き延ばすことが可能です。

「macOSと同等の極限の省電力性」を期待するのは酷ですが、Arch Linuxのカスタマイズ性とHyprlandの極上の操作感を、かつてのフラッグシップである大画面MacBook Proで体験できるメリットは非常に大きいです。使わなくなったIntel Macをお持ちの方は、ぜひOmarchyを導入して、自分だけの超高速な開発環境を構築してみてはいかがでしょうか。
